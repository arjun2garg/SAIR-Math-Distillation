═══ STEP 1 — PARSE BOTH EQUATIONS (write this out as your FIRST action) ═══

For EACH of the four sides — Eq1.LHS, Eq1.RHS, Eq2.LHS, Eq2.RHS — write a
single PARSE line in EXACTLY this format. Do not skip this step. Do not
shortcut. The rule layer below depends on these values being on the page.

  PARSE <side>: shape=<expr>  vars={...}  op=<n>  bare=<Y/N>
                lm=<v>  rm=<v>  ldepth=<n>  rdepth=<n>
                left_spine=<Y/N>  right_spine=<Y/N>

Field definitions (compute each by walking the expression as a tree
where every * is a binary node):
  shape       = the side, fully parenthesized exactly as written
  vars        = the SET of distinct variable letters appearing on the side
  op          = the count of * symbols on the side
  bare        = Y iff the side is a single variable letter; else N
  lm          = leftmost variable letter (read L→R, take first letter)
  rm          = rightmost variable letter (read L→R, take last letter)
  ldepth      = walk root→left child→left child→… until you hit a leaf;
                count how many * nodes you traversed
  rdepth      = same on the right-child path
  left_spine  = Y iff every * on the L-spine has the same leaf at the
                deepest left position AND no * appears off the L-spine.
                Concretely: ((((v * a) * b) * c)…) shape with v at the
                deepest left leaf and a, b, c, … each NOT containing *.
  right_spine = mirror of left_spine on the right (((…(c * (b * (a * v))))).

If a side is bare (single variable v), use: shape=v vars={v} op=0 bare=Y
lm=v rm=v ldepth=0 rdepth=0 left_spine=N right_spine=N.

Worked parse example for the side  x*(y*x):
  shape=(x*(y*x))  vars={x,y}  op=2  bare=N  lm=x  rm=x
  ldepth=1  rdepth=2  left_spine=N  right_spine=N
  (root is *; left child x is a leaf so ldepth=1; right child y*x is *
   whose right child x is a leaf so rdepth=2; off-spine * exists so
   neither pure spine.)
